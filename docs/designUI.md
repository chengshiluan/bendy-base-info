# Design UI

## 文档目的

这份文档用于约束管理类页面的 UI 与交互实现，避免后续开发再次出现下面这些问题：

- 搜索区和筛选区断成多行，视觉不齐
- 页面一进入就是编辑态，没有先展示表格
- 抽屉里直接堆表单，缺少列表层
- 操作入口散落在页面顶部，和数据行脱节
- 输入框、按钮、边距过紧，缺少现代感和秩序感

这不是“建议”，而是后续管理类页面默认执行标准。

## 适用范围

适用于所有后台管理页，尤其是：

- 账号管理
- 平台管理
- 注册源管理
- 密钥管理
- 密保管理
- 补充信息管理
- 其他结构相近的列表维护页

## 核心理念

### 1. 先展示，再编辑

- 用户进入页面后，第一眼应该看到可浏览的数据表格，不应该直接掉进编辑表单。
- 编辑是二级动作，必须通过数据行上的 `编辑`、`绑定`、`新增` 等入口进入。

### 2. 表格优先，不做卡片堆砌

- 结构化数据优先使用表格展示。
- 即使没有数据，也要正常展示表头。
- 空数据区域统一显示：`暂无数据`。

### 3. 操作跟着数据走

- 平台、注册源、密钥、密保、补充信息这类能力，不再通过页面顶部一堆独立按钮暴露。
- 如果某条数据缺少关联项，直接在对应单元格显示 `绑定`。
- 如果已有数据，在对应数据行提供 `编辑`、`绑定`、`删除` 等操作。

### 4. 编辑用弹窗，列表用抽屉

- 抽屉负责承载“当前上下文下的列表管理”。
- 弹窗负责承载“新增 / 编辑表单”。
- 禁止用户一进入抽屉就看到表单编辑态。

## 强制规范

## 工具栏规范

- 搜索框与筛选器必须融合成一个整体容器。
- 整个工具栏必须在同一视觉基线，禁止出现筛选器掉到第二行、按钮漂到另一侧中线不齐的情况。
- 搜索框、筛选器、按钮高度必须统一，当前标准为 `h-10`。
- 搜索区默认结构如下：

```text
[搜索输入框 | 状态 | 属性 | 置信度]                       [搜索] [新增]
```

- `状态`、`属性`、`置信度` 下拉框默认文案必须使用简洁名词，不要写成 `全部状态`、`全部属性`、`全部置信度`。
- 下拉面板中的第一项可以是 `全部`，但触发器默认显示必须是 `状态`、`属性`、`置信度`。
- 搜索按钮和新增按钮固定放在最右边，和左侧工具容器同高度。
- 禁止把搜索框、筛选器、搜索按钮、创建按钮拆成上下两排。

## 表格规范

- 每个管理抽屉必须默认展示表格。
- 表格列头必须始终可见。
- 操作列固定放在最右侧。
- 数据不足时也不能把表格替换成纯文案卡片。
- 空态统一使用表格中的一行文案：`暂无数据`。
- 表格应支持：
  - 单选
  - 全选
  - 批量删除
- 如果列较多，优先允许区域横向滚动或放大容器，不允许挤压到结构变形。

## 行为规范

- 进入 `平台`、`注册源`、`密钥`、`密保`、`补充信息` 时，默认必须看到“列表态”。
- 新增通过顶部右侧 `新增` 按钮进入弹窗。
- 编辑通过对应行的 `编辑` 按钮进入弹窗。
- 删除支持：
  - 行内单条删除
  - 勾选后的批量删除
- 缺失关联数据时，单元格展示 `绑定`，而不是在页面顶部再额外放一个直白按钮。

## 抽屉与弹窗规范

- 抽屉标题要简洁，优先使用：
  - `平台`
  - `注册源`
  - `密钥`
  - `密保`
  - `补充信息`
- 除非上下文必须说明，不要在标题里写成 `平台详情`、`注册源管理` 这种又长又重的标题。
- 编辑动作统一使用弹窗，不再在抽屉中直接编辑。
- 当抽屉承载的是宽表格时，抽屉必须支持可伸缩宽度。
- 抽屉负责上下文浏览，弹窗负责输入和提交，这个分层不能混。

## 表单美学规范

- 输入框不能紧贴边界，必须保留舒适留白。
- 默认推荐：
  - 输入框 `rounded-xl`
  - 多行输入 `rounded-2xl`
  - 左右内边距 `px-4`
  - 多行输入上下内边距 `py-3`
- 弹窗内容区域使用清晰的分段结构，不要把所有字段直接挤在一起。
- 底部操作区使用顶部分隔线，与表单主体拉开层次。

## 按钮状态规范

- 按钮在获得焦点或按压时可以出现高亮或白色态。
- 鼠标抬起、按钮失焦后，必须恢复正常颜色。
- 禁止出现“点了一下后一直保持白色”这种残留状态。

## 文案规范

- 页面与抽屉标题要短，不要堆说明词。
- 行为按钮优先使用短动词：
  - `搜索`
  - `新增`
  - `绑定`
  - `编辑`
  - `删除`
- 不要在局部按钮上堆砌“管理”“详情”“维护”等冗余词。

## 开发时禁止事项

- 禁止进入抽屉直接展示编辑表单。
- 禁止页面顶部堆多个“新增平台”“注册源管理”这种大而直白的入口。
- 禁止表格空态时隐藏表头。
- 禁止按钮高度不统一。
- 禁止搜索框、筛选器、按钮彼此断层错位。
- 禁止输入框和弹窗边界贴得太近。
- 禁止为了省事，把结构化数据渲染成多张卡片代替表格。

## 验收清单

每次做完管理类页面，至少自检以下项目：

- 顶部搜索区是否为一体化单行结构
- 搜索框、筛选器、按钮是否同高
- 默认进入是否先展示表格
- 空数据时是否显示表头和 `暂无数据`
- 是否支持单选 / 全选 / 批量删除
- 编辑是否通过弹窗完成
- 宽表格抽屉是否支持伸缩或足够宽
- 按钮点击后失焦是否恢复颜色
- 文案是否足够简洁
- 页面整体是否留白充足、边界不拥挤
