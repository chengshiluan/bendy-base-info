# Plan

## 使用规则

- 本文件是项目迭代计划主文件。
- 每次较大的开发任务开始前，先拆成可执行子任务，再写入本文件。
- 每完成一个子任务，就把对应的 `- [ ]` 更新为 `- [x]`。
- 本轮范围有变化时，必须同步更新本文件，避免计划与实际开发脱节。
- 根目录 `Plan.md` 只保留兼容入口，以 `docs/PLAN.md` 为准。

## 目标

把当前项目稳定沉淀为一个可持续维护的后台系统底座，优先完成：

- 可部署到 Vercel
- 可在 Neon / Upstash / S3 体系下稳定运行
- 可支持工作区、团队、用户、角色、权限、通知、工单、看板的持续迭代
- 可让后续开发按明确路线图推进，而不是临时补洞

## 协作规则

- 计划主文件使用 `docs/PLAN.md`
- 根目录 `Plan.md` 只保留兼容入口，不再维护详细内容
- 每完成一个可感知版本，更新 `CHANGELOG.md`
- 涉及运行、部署、排障、维护的内容，统一更新 `docs/maintain.md`
- 每次首次开始项目迭代，先阅读 `docs/` 目录下的 Markdown 文档
- 每次开发完成后默认执行 `git commit` 并 `git push origin <current-branch>`
- 默认先做“简单可用”，再做抽象与平台化
